## Phase Goal (user story)

**Как** пользователь iOS, **я хочу** нажать Connect и увидеть статусы и логи из нативного Swift-слоя, а туннель — стартовать и останавливаться через реальный Network Extension таргет, **чтобы** приложение работало на iOS так же, как на Android, и было готово к прогону через TestFlight.
> Сформулировано из ROADMAP Phase 5 Goal + Success Criteria (не выдумано).

## Этот план

Создать второй нативный таргет `PacketTunnel` (app-extension с подклассом `NEPacketTunnelProvider`) и настроить entitlements + App Group у обоих таргетов. Это фундамент вертикального среза: после плана в проекте есть исполняемый (пока покоящийся) NE-процесс, который поднимает `NEPacketTunnelNetworkSettings` с узким маршрутом-паритетом Android, и оба таргета собираются одной командой.

Purpose: без таргета `PacketTunnel` приложение не может стартовать реальный туннель (iOS исполняет код туннеля ТОЛЬКО в отдельном NE-процессе), а без entitlements/App Group у обоих таргетов подпись и shared-контейнер не работают. IOS-02 (провайдер туннеля) и IOS-04 (capabilities/entitlements/App Groups) закрываются здесь на уровне компиляции; реальный старт — device-checkpoint 05-03.

Output: `ios/PacketTunnel/` (провайдер, Info.plist, entitlements), `ios/Runner/Runner.entitlements`, коммит-скрипт `scripts/add_packet_tunnel_target.rb`, изменённый `project.pbxproj` с таргетом + embed-фазой + привязкой entitlements.
</objective>
