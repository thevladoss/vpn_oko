    - коннектор, который успешно «подключается» (возвращает Future без ошибки после короткой паузы) → measure() даёт LatencyMeasured, rtt.inMilliseconds >= 0
    - коннектор бросает SocketException('refused') → measure() даёт LatencyUnreachable (без rethrow)
    - measure(host, port) передаёт коннектору те же host и port и заданный timeout (проверить захваченные аргументы)
  </behavior>
  <action>
    Определи в файле пробы `typedef TcpConnector = Future<void> Function(String host, int port, Duration timeout);`.
    `class SocketLatencyProbe implements LatencyProbe`: конструктор `SocketLatencyProbe({TcpConnector? connector, this.timeout = const Duration(seconds: 3)})`; поле `_connect` = переданный connector или дефолт. Дефолтный коннектор: `Socket.connect(host, port, timeout: timeout)` затем `socket.destroy()` (никаких данных в сокет не пишем). `measure(host, port)`: старт `Stopwatch`, `await _connect(host, port, timeout)` в try → stop → `LatencyMeasured(Duration(milliseconds: sw.elapsedMilliseconds))`; `on SocketException` → `LatencyUnreachable()`. Инъекция коннектора — единственный шов для теста; реальный Socket.connect в автосьюте не вызывается (A3).
    Импорт `dart:io` только здесь (data-слой). Без комментариев.
