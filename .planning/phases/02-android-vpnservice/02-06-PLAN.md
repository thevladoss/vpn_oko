    Предусловия (окружение):
      0a. Запустить AVD API 34/35/36 (Android Studio → Device Manager). systemExempted и FGS-краш видны только на 34+.
      0b. adb доступен (`~/Library/Android/sdk/platform-tools/adb`, если не в PATH).
      0c. Автогейт перед ручной проверкой: `flutter analyze && flutter test && flutter build apk --debug` — зелёный; `cd android && ./gradlew :app:testDebugUnitTest` — зелёный.
      0d. Установить и запустить: `flutter run` (или `flutter install`). Приложение стартует без краша (контроль A3: тема FlutterFragmentActivity ок).

    Чеклист (нумерация из 02-VALIDATION.md):
      1. Тап Echo Connect → системный consent-диалог prepare() → «Разрешить» → статус Connected, значок ключа VPN в статус-баре. [AND-01, AND-02]
      2. В Connected открыть сайт в браузере эмулятора — грузится (узкий маршрут держит интернет). Если DNS/браузинг лёг (Assumption A1) — зафиксировать дефект: убрать/сменить addDnsServer. [AND-02]
      3. Уведомление foreground-сервиса видно в шторке (POST_NOTIFICATIONS выдан). [AND-03]
      4. `adb shell ping 10.111.222.1` → счётчик rx в harness растёт раз в секунду. [AND-05]
      5. Повторный Connect, в consent-диалоге нажать Cancel → статус Error + лог «VPN permission denied» (error-уровень). [AND-01]
      6. В Connected включить второй VPN поверх (или Settings → отозвать VPN) → onRevoke → статус Disconnected + лог «revoked by system». [AND-04]
      7. Цикл Connect→Disconnect→Connect ×3 — стабильно, уведомление снимается на Disconnect, fd не течёт (нет накопления потоков в logcat). [AND-03, AND-04]
      8. Нет краша на старте сервиса: в logcat отсутствуют ForegroundServiceDidNotStartInTimeException и MissingForegroundServiceTypeException. [AND-03]
      9. В логах harness виден переход состояний (state -> CONNECTING/CONNECTED/DISCONNECTING/DISCONNECTED) на каждом действии. [AND-06]

    Отметить каждый пункт pass/fail. Любой fail → описать наблюдение для gap-closure (`/gsd:plan-phase 2 --gaps`).
