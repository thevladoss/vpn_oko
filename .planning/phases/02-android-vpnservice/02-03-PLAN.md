    Поля: `rx = AtomicLong(0)`, `running = AtomicBoolean(false)`, `readThread: Thread?`, `trafficTicker: ScheduledExecutorService?` (или ScheduledFuture) — на выбор исполнителя (Discretion: механизм 1 Гц).
    `startReadLoop(pfd)`: `running.set(true)`, daemon-Thread читает `FileInputStream(pfd.fileDescriptor)` в один переиспользуемый `ByteArray(32767)` в цикле `while (running.get())`: `val n = input.read(buffer); if (n <= 0) break; rx.addAndGet(n.toLong())` — пакет ДРОПАЕТСЯ (форвардинга нет, вне скоупа). `IOException` при close() в teardown — ожидаема, проглотить.
    Ticker раз в секунду эмитит `VpnEventBus.emit(TrafficChangedMessage(rxBytes = rx.get(), txBytes = 0L))` (rx = байты из TUN = исходящий трафик устройства; txBytes=0, честно в README). Ticker может тикать с любого потока — шина потокобезопасна (план 01).
    `@Synchronized private fun teardown(reason: String)`: если `state` уже Disconnected — выйти (идемпотентность); `transition(Disconnecting)`; `running.set(false)`; `tunnel?.close(); tunnel = null` (close ДО join — разблокирует блокирующий read, Pitfall 7); `readThread?.join(500); readThread = null`; остановить ticker; `VpnEventBus.emit(LogMessage(reason, now, "info"))`; `ServiceCompat.stopForeground(this, ServiceCompat.STOP_FOREGROUND_REMOVE)`; `transition(Disconnected)`; `stopSelf()`. (transition сам гейтит через canTransition: из Error путь Disconnecting отклонится с error-логом, но финальный transition(Disconnected) допустим и доведёт статус.)
    `override fun onRevoke() { teardown("revoked by system") }` — приходит НЕ на main thread, события идут через шину (Pitfall 6).
    `override fun onDestroy() { tunnel?.close(); tunnel = null; super.onDestroy() }`.
    Без комментариев.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Activity Intent extras → Service | host/port/userId приходят из Intent (потенциально невалидны) |
| TUN пакеты → read-loop | входящие байты из туннеля |
| система (onRevoke) → Service | отзыв VPN другим приложением, не на main thread |
